| Observação | O analista apenas observa o usuário trabalhando sem interferirQuando o processo já é definido e se quer ver o fluxo real.Não interrompe o trabalho e mostra o processo natural.Não explica o motivo de algumas ações.O analista observa a recepcionista trabalhando e percebe que ela usa vários sistemas para buscar o convênio. | O analista participa das atividades junto com o usuário. O analista marca se cada etapa foi concluída, com erro ou tempo gasto. | Registro do processo por gravação para análise posterior. Quando o processo é complexo e precisa ser revisado.Permite analisar detalhes depois.Pode deixar o usuário desconfortável e exige autorização.O analista grava a tela para ver onde o usuário clica mais ou encontra dificuldades. |
| Análise de Documentos | Técnica de levantamento de requisitos que analisa documentos existentes da organização para entender processos e regras. Quando a organização possui muitos registros formais sobre como o trabalho funciona.Acesso a informações detalhadas sem interromper o trabalho dos usuários.Documentos podem estar desatualizados ou incompletos.O analista analisa manuais, formulários, relatórios e regulamentos para identificar regras, dados e etapas do processo. | Materiais que descrevem processos e regras da organização. Quando é necessário entender como o sistema ou processo atual funciona.Ajuda a identificar dados e regras de negócio importantes.Pode não refletir exatamente a prática real.Analisar manuais, formulários, relatórios administrativos e regulamentos internos. | Processo de identificar regras, dados e funcionalidades a partir dos documentos.
Quando se deseja transformar informações existentes em requisitos do sistema.Facilita a definição de funcionalidades necessárias.Nem sempre mostra todas as necessidades dos usuários.O analista lê documentos e identifica processos, dados usados e regras de negócio que o sistema deve atender. | Documentos formais que descrevem exigências para desenvolvimento de sistemas. Projetos públicos ou contratos que exigem requisitos definidos.Apresentam regras e funcionalidades obrigatórias do sistema.Podem ser muito técnicos ou genéricos.O analista lê o edital e identifica exigências, normas e funcionalidades que o sistema deve possuir. | Registro das atividades do usuário por gravação para análise posterior. Quando é necessário revisar processos ou analisar tarefas complexas.Permite observar detalhes do processo várias vezes.Pode deixar o usuário desconfortável e exige autorização.O analista grava áudio, vídeo ou a tela enquanto o usuário realiza uma tarefa para analisar depois. |
